# 🧀 JerryKeyring 2026 프로젝트 마스터 가이드 (Project Blueprint)

이 문서는 JerryKeyring 프로젝트의 핵심 아키텍처, 데이터베이스 구조, UI 정의 및 작업 원칙을 정리한 마스터 가이드입니다. 모든 개발 작업은 이 문서의 원칙을 준수해야 합니다.

---

## 1. 프로젝트 개요
JerryKeyring은 NFC 키링을 활용하여 개인화된 음악 감상 경험을 제공하는 웹 서비스입니다. 레거시 시스템의 감성을 유지하면서 Supabase와 Cloudflare R2를 활용하여 현대적인 인프라로 고도화되었습니다.

---

## 2. 데이터베이스 구조 (Supabase)

### 🔹 `tracks` 테이블
유저의 마스터 정보와 곡의 메타데이터를 보관합니다.
- `nfc_id` (PK): NFC 태그 고유 식별자 (예: 9220)
- `genre`: 곡의 장르 (콤마로 구분하여 다중 트랙 지원)
- `status`: 현재 상태 (주로 가사 폴백용으로 사용)
- `lyrics`: 기본 가사
- `sync_lyrics`: [mm:ss.00] 타임스탬프가 포함된 싱크 가사
- `story`: AI 라디오 생성을 위한 유저 사연
- `serial`: 기기 시리얼 번호 (SN: XXXX)
- `music_title`: (신규 추가) 사용자가 직접 기입한 곡의 제목

### 🔹 `audio_files` 테이블
실제 음원 파일의 저장소 경로(Cloudflare R2)를 관리합니다.
- `nfc_id`: 연동된 유저의 식별자
- `audio_url`: Cloudflare R2에 업로드된 실제 mp3 주소
- `category`: 파일 유형 (`music`, `radio_tts`)
- `radio_script`: TTS 생성을 위한 라디오 대본

---

## 3. UI/UX 영역 정의

### 📱 메인 화면 (Index/Player 초기화면)
- **목적**: 유저가 보유한 음원 목록을 버튼 형태로 노출.
- **기능**: 버튼 클릭 시 해당 곡 재생 시작.

### 🎵 감상 모드 (OBS Mode)
- **목적**: 음악 감상에 몰입할 수 있는 시각적 경험 제공.
- **기능**: 유휴 시간(5초) 발생 시 UI 숨김, 큰 시계 및 가사 표시.

### ⚙️ 플레이어 컨트롤
- **목적**: 재생 제어 및 개인화 설정.
- **기능**: 재생/일시정지, 이전/다음 곡, 가사 보기, 환경 설정.

## 4. 작업 원칙 및 협업 로직 (중요)

1.  **기존 코드 보존**: 새로운 기능을 추가할 때 기존에 잘 작동하던 로직을 절대 해치지 않습니다. 수정보다는 **기존 코드 기반의 확장**을 우선합니다.
2.  **폴백 로직 필수**: DB에 음원 정보가 없더라도 로컬 파일(`./music/ID.mp3`)을 통해 재생이 가능하도록 이중 안전장치를 유지합니다.
3.  **검증 후 푸시**: 모든 솔루션 적용 후 반드시 `git push`를 수행합니다.
4.  **수동 푸시 대응**: 로컬 환경 이슈로 푸시가 실패할 경우, 사용자에게 수동 푸시 명령어를 안내합니다.
5.  **브랜치 전략**: 
    *   `main`: 실제 서비스 중인 안정적인 브랜치.
    *   `v0`: 2026-04-26 시점의 가장 안정적인 상태를 박제한 레거시 보존 브랜치.
