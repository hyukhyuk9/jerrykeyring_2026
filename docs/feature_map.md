# 🗺️ 서비스 기능 및 페이지 맵 (Feature & Page Map)

이 문서는 JerryKeyring 2026 프로젝트의 모든 화면 구성과 각 화면별 상세 기능을 정의합니다.

---

## 1. 유저 사이드 (User Experience)

NFC 태그를 통해 접속하는 일반 사용자용 웹 환경입니다.

### 📱 메인 플레이어 페이지 (`/player/[nfc_id]`)
*   **음원 재생**: 태그에 할당된 메인 트랙 재생 (Visualizer 포함).
*   **가사 디스플레이**: 일반 가사 및 타임스탬프 기반 싱크 가사 노출.
*   **AI 라디오 청취**: 유저 사연 기반으로 생성된 AI 도슨트/라디오 음원 재생 버튼.
*   **개인화 영역**: 곡 제목, 장르, 유저 메시지 시각화.
*   **OBS 몰입 모드**: 5초 이상 유휴 시 UI가 사라지고 비주얼만 남는 모드 전환.

---

## 2. 관리자 사이드 (Admin Dashboard)

운영자가 시스템 상태를 점검하고 데이터를 관리하는 대시보드입니다.

### 📊 대시보드 홈 (`/admin`)
*   **시스템 요약**: 총 활성 태그 수, 오늘 접속 수, AI 생성 대기 건수 요약.
*   **이슈 알림**: 음원 파일 누락(404)이나 AI 생성 실패 건 실시간 경고.

### 🏷️ 태그 및 인벤토리 관리 (`/admin/tags`)
*   **태그 등록**: 새로운 NFC ID 및 시리얼 번호 DB 등록.
*   **상태 제어**: 특정 태그의 활성화/비활성화(분실 처리 등) 상태 변경.

### 🎵 트랙 및 사용자 관리 (`/admin/tracks`)
*   **데이터 수정**: 유저 이름, 연락처, 곡 제목, 사연, 가사 직접 수정.
*   **AI 라디오 제어**: 
    *   사연을 바탕으로 라디오 스크립트 자동 생성 요청.
    *   생성된 스크립트로 TTS 음원 파일 빌드 실행.
*   **매핑 관리**: 특정 NFC ID에 다른 음원 파일을 수동으로 연결.

### 📁 스토리지 및 자원 모니터링 (`/admin/resources`)
*   **R2 상태 체크**: Cloudflare R2에 업로드된 파일들의 실제 존재 여부 전수 조사.
*   **로그 확인**: `access_logs` 기반의 유저 접속 트래픽 시각화.

---

## 3. 핵심 공통 기능 (Core Logic)

*   **NFC 인증 (Auth)**: 특정 토큰이나 시리얼 검증을 통한 비인가 접근 차단.
*   **다국어/다중 장르**: 콤마로 구분된 장르 데이터를 기반으로 한 동적 UI 테마 적용.
*   **자동 복구 (Fallback)**: 네트워크 불안정 시 로컬 폴백 음원 재생 시도.
