# 🔑 카카오 로그인 및 월간 무료 토큰 시스템 기획안

이 문서는 사용자 리텐션 향상을 위한 카카오 로그인 연동 및 월간 무료 음원 제작 토큰(Credit) 시스템 도입을 위한 가이드입니다.

---

## 1. 시스템 아키텍처
- **인증**: Supabase Auth (OAuth 2.0 - Kakao)
- **데이터베이스**: Supabase PostgreSQL
- **프론트엔드**: Next.js / Vanilla JS (Legacy Player)

## 2. 데이터베이스 설계 (Supabase)

### `profiles` 테이블
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | uuid | `auth.users.id`와 연동 (PK) |
| `email` | text | 사용자 이메일 |
| `nickname` | text | 카카오 닉네임 |
| `credits` | integer | 남은 무료 제작 횟수 (기본값: 1) |
| `last_refill_date` | timestamp | 마지막 토큰 충전 날짜 |

## 4. 핵심 로직 플로우

1. **로그인 시점 (Refill Check)**
   - 사용자가 로그인하면 `last_refill_date`를 조회합니다.
   - 현재 날짜가 마지막 충전일보다 새로운 '달(Month)'이라면 `credits`를 1로 초기화하고 날짜를 업데이트합니다.

2. **곡 제작 시점 (Credit Consumption)**
   - 제작 버튼 클릭 시 `credits > 0` 인지 확인합니다.
   - 제작 성공 시 `credits`를 1 차감합니다.

3. **리텐션 유도 (Notification)**
   - 토큰이 충전되었을 때 알림톡 또는 푸시를 발송하여 재방문을 유도합니다.
